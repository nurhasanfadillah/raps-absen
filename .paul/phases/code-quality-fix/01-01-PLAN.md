<task type="auto">
  <name>Task 1: Install dan konfigurasi ESLint + Prettier</name>
  <files>package.json, .eslintrc.json, .prettierrc</files>
  <action>
    1. Install dev dependencies:
       ```
       npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin eslint-plugin-react eslint-plugin-react-hooks prettier eslint-config-prettier
       ```

    2. Buat `.eslintrc.json`:
       ```json
       {
         "parser": "@typescript-eslint/parser",
         "parserOptions": { "ecmaVersion": 2022, "sourceType": "module" },
         "plugins": ["@typescript-eslint", "react", "react-hooks"],
         "extends": [
           "eslint:recommended",
           "plugin:@typescript-eslint/recommended",
           "plugin:react-hooks/recommended",
           "prettier"
         ],
         "rules": {
           "@typescript-eslint/no-explicit-any": "warn",
           "@typescript-eslint/no-unused-vars": ["warn", { "argsIgnorePattern": "^_" }],
           "react-hooks/exhaustive-deps": "warn"
         },
         "env": { "browser": true, "es2022": true }
       }
       ```

    3. Buat `.prettierrc`:
       ```json
       {
         "singleQuote": true,
         "trailingComma": "es5",
         "semi": true,
         "tabWidth": 2,
         "printWidth": 100
       }
       ```

    4. Tambah scripts ke `package.json`:
       ```json
       "lint": "eslint . --ext .ts,.tsx --ignore-path .gitignore",
       "format": "prettier --write \"**/*.{ts,tsx,json}\" --ignore-path .gitignore"
       ```

    Hindari: jangan set `@typescript-eslint/no-explicit-any` ke "error" — terlalu banyak existing violations, jadikan "warn" dulu.
  </action>
  <verify>Jalankan `npm run lint` — harus keluar tanpa crash (warning OK). Jalankan `npm run format` — harus memformat file tanpa error.</verify>
  <done>AC-1 dan AC-2 terpenuhi: lint dan format berjalan</done>
</task>

<task type="auto">
  <name>Task 2: Aktifkan TypeScript strict mode dan perbaiki type errors</name>
  <files>tsconfig.json, store.tsx, App.tsx, pages/Employees.tsx, pages/Payroll.tsx, pages/CashAdvance.tsx, pages/Settings.tsx, pages/Dashboard.tsx, pages/ActivityLogs.tsx</files>
  <action>
    1. Di `tsconfig.json`, tambah ke `compilerOptions`:
       ```json
       "strict": true,
       "noUnusedLocals": true,
       "noUnusedParameters": false
       ```
       `noUnusedParameters: false` karena React event handlers sering punya unused params.

    2. Jalankan `npx tsc --noEmit` untuk melihat semua errors.

    3. Perbaiki errors yang muncul — yang paling umum:
       - **Possibly null/undefined**: Tambah optional chaining (`?.`) atau null guard
       - **Missing return type**: Tambah return type annotation jika TypeScript minta
       - **Implicit any pada callback parameter**: Beri type annotation eksplisit

    4. Untuk `store.tsx` — explicit `any` pada data mapping (`(e: any)`, `(a: any)`, dll) akan muncul sebagai ESLint warning tapi BUKAN TypeScript error karena sudah explicit. Biarkan dulu, akan diaddress di plan arsitektur.

    5. Untuk `components/Layout.tsx:1` — hapus unused `useRef` import.

    Hindari: jangan tambah `// @ts-ignore` atau `// @ts-expect-error` sebagai jalan pintas — fix type error sesungguhnya.
  </action>
  <verify>Jalankan `npx tsc --noEmit` — harus output `0 errors`. Jalankan `npm run build` — harus berhasil.</verify>
  <done>AC-3 terpenuhi: TypeScript strict mode aktif, 0 compile errors</done>
</task>
