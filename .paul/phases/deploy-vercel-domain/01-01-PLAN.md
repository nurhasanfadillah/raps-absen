<objective>
## Goal
Deploy aplikasi React + Vite ke Vercel production via CLI dan hubungkan domain `raps.redone.my.id`.

## Purpose
Membuat aplikasi RAPS dapat diakses publik melalui domain kustom dengan konfigurasi deploy yang proper untuk SPA.

## Output
- `vercel.json` — SPA routing config
- `vite.config.ts` — dibersihkan dari define block yang tidak diperlukan
- App live di Vercel production URL
- Domain `raps.redone.my.id` terhubung ke Vercel
- Instruksi DNS untuk dikonfigurasi di DNS panel
</objective>

<tasks>

<task type="auto">
  <name>Task 1: Buat vercel.json dan bersihkan vite.config.ts</name>
  <files>vercel.json, vite.config.ts</files>
  <action>
    1. Buat file `vercel.json` di root project dengan isi:
       {
         "rewrites": [
           { "source": "/(.*)", "destination": "/index.html" }
         ]
       }

    2. Edit `vite.config.ts`:
       - Hapus import `loadEnv` dari 'vite'
       - Ubah signature dari `defineConfig(({ mode }) => { const env = loadEnv(...); return {...} })` 
         menjadi `defineConfig({...})` langsung (object, bukan function)
       - Hapus seluruh block `define` yang berisi GEMINI_API_KEY
       - Pertahankan: plugins, server config, dan resolve alias

    Hasil akhir vite.config.ts:
    ```ts
    import path from 'path';
    import { defineConfig } from 'vite';
    import react from '@vitejs/plugin-react';

    export default defineConfig({
      server: {
        port: 3000,
        host: '0.0.0.0',
      },
      plugins: [react()],
      resolve: {
        alias: {
          '@': path.resolve(__dirname, '.'),
        }
      }
    });
    ```
  </action>
  <verify>
    - Baca vercel.json: harus ada rewrites rule source "/(.*)" → "/index.html"
    - Baca vite.config.ts: tidak ada kata "loadEnv", "GEMINI_API_KEY", atau "define"
  </verify>
  <done>AC-1 dan AC-2 terpenuhi: vercel.json ada, vite.config.ts bersih.</done>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Login ke Vercel dan deploy ke production</action>
  <instructions>
    Jalankan perintah berikut di terminal (dari root project):

    **Step 1 — Login Vercel (sekali saja, skip jika sudah login):**
    ```
    npx vercel login
    ```
    Pilih metode login (GitHub direkomendasikan), browser akan terbuka untuk autentikasi.

    **Step 2 — Deploy ke production:**
    ```
    npx vercel --prod
    ```
    Saat pertama kali, CLI akan tanya:
    - "Set up and deploy?" → Y
    - "Which scope?" → pilih akun kamu
    - "Link to existing project?" → N (project baru)
    - "Project name?" → raps (atau terserah)
    - "In which directory is your code?" → . (titik, berarti root)
    - Framework akan auto-detect sebagai Vite
    - Build command: `npm run build` (sudah benar)
    - Output dir: `dist` (sudah benar)

    Tunggu hingga deployment selesai. Catat URL production yang diberikan
    (format: https://raps-xxx.vercel.app)
  </instructions>
  <resume-signal>Ketik URL production Vercel kamu (contoh: https://raps-abc.vercel.app) untuk lanjut</resume-signal>
</task>

<task type="checkpoint:human-action" gate="blocking">
  <action>Tambah custom domain dan konfigurasi DNS</action>
  <instructions>
    **Step 1 — Tambah domain ke Vercel via CLI:**
    ```
    npx vercel domains add raps.redone.my.id
    ```

    Vercel akan menampilkan DNS record yang perlu ditambahkan.
    Biasanya berupa CNAME record seperti:
    - Type: CNAME
    - Name: raps
    - Value: cname.vercel-dns.com

    **Step 2 — Buka DNS panel untuk redone.my.id** di provider domain kamu
    (Cloudflare / Niagahoster / dll), lalu tambahkan record sesuai output CLI tadi.

    **Step 3 — Verifikasi propagasi DNS (tunggu 1–10 menit):**
    ```
    npx vercel domains inspect raps.redone.my.id
    ```
    Status harus "Valid Configuration" sebelum domain aktif.
  </instructions>
  <resume-signal>Ketik "domain aktif" setelah vercel domains inspect menunjukkan Valid Configuration</resume-signal>
</task>
