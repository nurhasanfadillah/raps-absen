<objective>
## Goal
Ubah default filter kasbon ke "pending" dan hapus blokir edit/hapus di store layer agar konsisten dengan perubahan UI fase sebelumnya.

## Purpose
Store masih melempar error untuk operasi edit/hapus pada kasbon lunas/terpotong, membuat tombol Edit/Hapus di UI tidak berfungsi meski sudah tampil.

## Output
`pages/CashAdvance.tsx` dan `store.tsx` diperbarui.
</objective>

<acceptance_criteria>

## AC-1: Default filter pending
```gherkin
Given user membuka halaman Kasbon
When halaman pertama kali dimuat
Then filter aktif adalah "Pending" (bukan "Semua")
  And hanya kasbon dengan isPaid=false yang tampil secara default
```

## AC-2: Edit kasbon lunas berfungsi
```gherkin
Given kasbon dengan isPaid=true atau deductedInPayrollId terisi
When user klik Edit dan submit form perubahan
Then perubahan tersimpan ke database tanpa error
  And audit log UPDATE tercatat di activity logs
```

## AC-3: Hapus kasbon lunas berfungsi
```gherkin
Given kasbon dengan isPaid=true atau deductedInPayrollId terisi
When user klik Hapus dan konfirmasi dialog
Then kasbon terhapus dari database tanpa error
  And audit log DELETE tercatat di activity logs
```
